Exercise 2.1 – Bootstrap Directory Structure

When you download Bootstrap 5, the archive extracts into this structure:

bootstrap-5.x.x-dist/
│
├── css/
│   ├── bootstrap.css              ← Full unminified CSS
│   ├── bootstrap.min.css          ← Minified CSS (use in production)
│   ├── bootstrap.css.map          ← Source map for debugging
│   ├── bootstrap-grid.css         ← Grid system only
│   ├── bootstrap-grid.min.css
│   ├── bootstrap-reboot.css       ← CSS reboot/normalize only
│   ├── bootstrap-reboot.min.css
│   ├── bootstrap-utilities.css    ← Utility classes only
│   └── bootstrap-utilities.min.css
│
└── js/
    ├── bootstrap.js               ← Full unminified JS (all plugins)
    ├── bootstrap.min.js           ← Minified JS
    ├── bootstrap.bundle.js        ← JS + Popper.js bundled
    ├── bootstrap.bundle.min.js    ← Minified bundle (recommended)
    ├── bootstrap.esm.js           ← ES Module version
    └── bootstrap.esm.min.js
css/ folder

Contains all compiled CSS. Includes:

  • bootstrap.min.css – main stylesheet to include in production
  • bootstrap-grid.css – only the grid, for lightweight projects
  • bootstrap-reboot.css – cross-browser baseline styles
  • bootstrap-utilities.css – utility classes only (margins, padding, flex, etc.)
  • *.map files – source maps for browser DevTools debugging
js/ folder

Contains all JavaScript plugins. Includes:

  • bootstrap.bundle.min.js – Bootstrap plugins + Popper.js; the recommended single include
  • bootstrap.min.js – Bootstrap plugins only (Popper must be added separately)
  • bootstrap.esm.min.js – ES Module build for modern bundlers (Vite, Webpack)
Bootstrap Icons (separate)

Bootstrap Icons is a separate package and not included in the main download:

  • Install via npm: npm i bootstrap-icons
  • Or use CDN: bootstrap-icons/font/bootstrap-icons.css
  • Contains 2000+ SVG and font icons
