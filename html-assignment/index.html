📅 July Events Now Open

Registration for all July community events is now live. Residents can sign up online through the portal's Registration section. Spots are limited for the Kids Art Workshop and Heritage Night Market — early registration is encouraged to avoid disappointment.

🏭 Park Renovations Complete

The City Park renovation project has been completed ahead of schedule. The new facilities include upgraded restrooms, improved lighting, a dedicated cycling path, and a brand new bandstand for the upcoming Summer Fest 2025. Residents are invited to the official ribbon-cutting ceremony on 5 July.

✎ What's New This Month

  • Free Wi-Fi installed at all community centres
  • New food market permit issued for Heritage Night Market
  • Youth coding workshop — register before 10 July

📌 Council Notice

The next City Council meeting will be held on 15 July at 7 pm in Town Hall. Residents are welcome to attend and raise questions during the open forum. Agenda items include the proposed cycle-lane extension and the 2026 events budget allocation. Written submissions accepted until 12 July.

JavaScript Exercises

All 14 exercises are live on this page. Open DevTools Console F12 to see detailed step-by-step logs from main.js.

Ex 1–2  ·  Basics, Data Types & Operators

JavaScript Setup, Variables & Seat Counter

const eventName, const eventDate — never reassigned. let availableSeats — updated on each click. Template literal builds the info string above. console.log("Welcome to the Community Portal") fires on script load — check DevTools.

Ex 3  ·  Conditionals, Loops & Error Handling

Event Status List — forEach · if-else · try-catch

    allEvents.forEach() loops every event; if-else decides the status label. The register button calls registerUser() which wraps registerWithErrorHandling() in a try-catch. Try clicking it multiple times to watch seats drop and errors fire.

    Ex 4  ·  Functions, Scope, Closures & Higher-Order

    addEvent() · registerUser() · filterEventsByCategory() · Closure Tracker

    Register via Ex 7–8 cards to see category counts here.

    createCategoryTracker() returns an object whose counts map is private to its closure scope. filterEventsByCategory(cat, callback) is a higher-order function — it receives and invokes a callback with the filtered results.

    Ex 5  ·  Objects & Prototypes

    Event Constructor · Prototype Method · Object.entries()

    checkAvailability() lives on EventObj.prototype — shared by every instance without copying it. Object.entries(protoEvent) returns own enumerable key–value pairs only (prototype methods excluded).

    Ex 6  ·  Arrays & Methods

    .push()  ·  .filter()  ·  .map()

    All operations run on a spread clone ([...allEvents]) so the live allEvents array is never mutated. Add events above and click the Refresh button to see the counts update.

    Ex 7–8  ·  DOM Manipulation & Event Handling

    Dynamic Event Cards — createElement · onchange · keydown · onclick

    Cards are built entirely via createElement() + appendChild(). "Register" uses onclick, the dropdown uses onchange, and the search field uses keydown. Cancelling a card removes it from the DOM.

    Ex 9  ·  Async JS — Promises & Async/Await

    Fetch Mock Event Data from API

    Fetching from API…

    Data is fetched from jsonplaceholder.typicode.com as a stand-in for a real events API. The spinner is shown while the request is in-flight and hidden on resolution or error. Both buttons hit the same endpoint — one uses .then(), the other await.

    Ex 11  ·  Working with Forms

    form.elements · preventDefault · Inline Validation

    Fill in the Registration Form above, then click below to read all inputs via form.elements and validate them.

    event.preventDefault() is called on form submit (see submitRegistration() in the inline script). Errors are shown inline next to fields. This button reads the live form state via form.elements['name'], etc.

    Ex 12  ·  AJAX & Fetch API (POST)

    POST Registration Data to Mock Server

    setTimeout() wraps a Promise to simulate a 1.5 s server round-trip before the actual fetch() POST fires. Content-Type: application/json is set in the request headers, and the mock server echoes back a generated id.

    Ex 13  ·  Debugging & Testing

    Form Submission Debug Trace — console.group · Breakpoint

    Runs a step-by-step grouped trace in the browser console showing captured form values, validation results, and the prepared fetch payload. Add a breakpoint in DevTools at main.js or uncomment the debugger; statement to pause execution mid-trace.

    Open the Network tab in DevTools and click POST via fetch() in Ex 12 to inspect the full request/response including headers and payload. Use Sources → main.js to set breakpoints.
